# Spectre Nixarr Media Automation Stack

Spectre runs the media automation stack through `nixarr`, using Dirtycow's `tank/media` export mounted at `/mnt/media`.

## Services

| Service | Purpose | URL | Backend |
| :--- | :--- | :--- | :--- |
| Seerr / Jellyseerr | Request UI for movies and TV | `https://request.burdznest.com` | `127.0.0.1:5055` |
| Radarr | Movie automation | `https://radarr.burdznest.com` | `127.0.0.1:7878` |
| Sonarr | TV automation | `https://sonarr.burdznest.com` | `127.0.0.1:8989` |
| Prowlarr | Indexer management | `https://prowlarr.burdznest.com` | `127.0.0.1:9696` |
| Bazarr | Subtitle automation | `https://bazarr.burdznest.com` | `127.0.0.1:6767` |
| qBittorrent / qui | Torrent client frontend | `https://torrent.burdznest.com` | `127.0.0.1:5252` |
| qBittorrent WebUI | Native qBittorrent admin UI | SSH tunnel only | `127.0.0.1:8085` |

All nixarr UIs are routed through Traefik with the `internal-only` middleware. They are only intended to be reachable from the trusted LAN ranges `10.0.0.0/24` and `10.0.1.0/24`.

## Storage Layout

Nixarr's media root is:

```bash
/mnt/media
```

This is the NFS mount from Dirtycow's `tank/media` dataset. Keep all media automation paths under this mount so Radarr, Sonarr, qBittorrent, Jellyfin, and request workflows see the same files.

Recommended first-run directories:

```bash
sudo mkdir -p /mnt/media/downloads/incomplete /mnt/media/movies /mnt/media/tv
sudo chown -R burdz:media /mnt/media/downloads /mnt/media/movies /mnt/media/tv
sudo chmod 2775 /mnt/media/downloads /mnt/media/downloads/incomplete /mnt/media/movies /mnt/media/tv
sudo find /mnt/media/downloads /mnt/media/movies /mnt/media/tv -type d -exec chmod g+s {} \;
```

These services write through the shared `media` group:

| Service | Runs as | Required writable path |
| :--- | :--- | :--- |
| Radarr | `radarr:media` | `/mnt/media/movies` |
| Sonarr | `sonarr:media` | `/mnt/media/tv` |
| qBittorrent | `qbittorrent:media` | `/mnt/media/downloads` and `/mnt/media/downloads/incomplete` |

Media directories should look like `drwxrwsr-x`: group `media`, group-writable, and setgid so new files inherit the `media` group. If `chmod` does not stick through NFS from Spectre, run the corresponding commands on Dirtycow under `/tank/media/...`.

Verify service write access from Spectre:

```bash
sudo -u radarr touch /mnt/media/movies/.radarr-test && sudo rm /mnt/media/movies/.radarr-test
sudo -u sonarr touch /mnt/media/tv/.sonarr-test && sudo rm /mnt/media/tv/.sonarr-test
sudo -u qbittorrent touch /mnt/media/downloads/incomplete/.qbit-test && sudo rm /mnt/media/downloads/incomplete/.qbit-test
```

Use these paths in the web UIs:

| Path | Use |
| :--- | :--- |
| `/mnt/media/downloads` | qBittorrent download location |
| `/mnt/media/downloads/incomplete` | Optional qBittorrent incomplete download location |
| `/mnt/media/movies` | Radarr root folder |
| `/mnt/media/tv` | Sonarr root folder |

## qBittorrent / qui setup

`https://torrent.burdznest.com` serves qui, the qBittorrent frontend. qui may not expose qBittorrent's default save path setting, so use the native qBittorrent WebUI over an SSH tunnel for initial download path setup.

1. **Open a tunnel to Spectre:**
   ```bash
   ssh -L 8085:127.0.0.1:8085 burdz@10.0.0.71
   ```

2. **Get the temporary qBittorrent admin password** from Spectre:
   ```bash
   journalctl -u qbittorrent -b | grep -i 'temporary password'
   ```

3. **Open the native WebUI locally:**
   ```text
   http://127.0.0.1:8085
   ```

4. **Set the save paths** in qBittorrent settings:
   - Default Save Path: `/mnt/media/downloads`
   - Optional incomplete path: `/mnt/media/downloads/incomplete`

After this is set, day-to-day torrent checks can use qui at `https://torrent.burdznest.com`.

## First-Run Debugging

### Sonarr Bad Gateway

If `https://sonarr.burdznest.com` returns Bad Gateway, Sonarr may look active in systemd but not be listening on `8989`. During first-run this happened because `/var/lib/nixarr/sonarr` was not writable by the service user.

The current Nix config includes tmpfiles state-directory ownership entries for nixarr services. To debug without changing config:

```bash
journalctl -u sonarr -b -n 80 --no-pager
curl -I http://127.0.0.1:8989
```

If the local `curl` fails and logs mention write/permission errors under `/var/lib/nixarr/sonarr`, verify the state directory ownership generated by the current tmpfiles rules.

### Stalled or errored torrents

qBittorrent systemd logs usually only show service health, not per-torrent failures. For stuck downloads, check the native qBittorrent WebUI torrent details:

- Trackers
- Peers
- General / Info
- Error

Public indexers can return dead, fake, or stale torrents even when they show seed counts. If a torrent stalls, try a manual search, a different release, better indexers, or Usenet later.

## VPN Status

qBittorrent is currently **not behind a VPN**.

Current state:

| Setting | Value |
| :--- | :--- |
| `nixarr.vpn.enable` | `false` |
| `nixarr.qbittorrent.vpn.enable` | `false` |
| qBittorrent peer port | `50000` |
| WireGuard config path | `/var/lib/nixarr/wg.conf` |

To add a VPN later:

1. Get a WireGuard config from the VPN provider.
2. Place it on Spectre at:
   ```bash
   sudo install -m 0600 -o root -g root wg.conf /var/lib/nixarr/wg.conf
   ```
3. Update the Spectre nixarr config to enable both the global VPN and qBittorrent VPN flags.
4. Replace `peerPort = 50000` with the forwarded VPN port, if the provider assigns one.
5. Rebuild Spectre and verify qBittorrent traffic exits through the VPN before adding indexers.

## First-Run Setup

1. **Verify mounts and services on Spectre:**
   ```bash
   findmnt /mnt/media
   systemctl status seerr radarr sonarr prowlarr bazarr qbittorrent
   ```

2. **Create or repair media folders and permissions** if needed:
   ```bash
   sudo mkdir -p /mnt/media/downloads/incomplete /mnt/media/movies /mnt/media/tv
   sudo chown -R burdz:media /mnt/media/downloads /mnt/media/movies /mnt/media/tv
   sudo chmod 2775 /mnt/media/downloads /mnt/media/downloads/incomplete /mnt/media/movies /mnt/media/tv
   sudo find /mnt/media/downloads /mnt/media/movies /mnt/media/tv -type d -exec chmod g+s {} \;
   ```

3. **Configure qBittorrent save paths** through the native WebUI SSH tunnel:
   - Default Save Path: `/mnt/media/downloads`
   - Optional incomplete path: `/mnt/media/downloads/incomplete`
   - Use `https://torrent.burdznest.com` for qui after the native setting is saved.

4. **Open Prowlarr** at `https://prowlarr.burdznest.com`:
   - Add indexers.
   - Confirm nixarr sync has Radarr and Sonarr applications configured.
   - Test each app connection.

5. **Open Radarr** at `https://radarr.burdznest.com`:
   - Set the movie root folder to `/mnt/media/movies`.
   - Confirm qBittorrent is available as the download client.

6. **Open Sonarr** at `https://sonarr.burdznest.com`:
   - Set the TV root folder to `/mnt/media/tv`.
   - Confirm qBittorrent is available as the download client.

7. **Open Bazarr** at `https://bazarr.burdznest.com`:
   - Confirm Radarr and Sonarr sync are connected.
   - Add subtitle providers.

8. **Open Seerr / Jellyseerr** at `https://request.burdznest.com`:
   - Connect Jellyfin.
   - Connect Radarr at `127.0.0.1:7878` with SSL off, URL base blank, username/password blank, and API key set.
   - Connect Sonarr at `127.0.0.1:8989` with SSL off, URL base blank, username/password blank, and API key set.
   - Use root folders `/mnt/media/movies` and `/mnt/media/tv`.
   - Submit a test request and verify it appears in the correct automation app.

9. **Add Jellyfin libraries** using the same media folders:
   - Movies: `/mnt/media/movies`
   - TV: `/mnt/media/tv`
